WebHook이란

웹 애플리케이션에서 발생하는 이벤트를 다른 애플리케이션으로 전달하는 방식.
이벤트 기반 아키텍처를 지원하며, 하나의 애플리케이션이 다른 애플리케이션의 상태 변경이나 사건을 실시간으로 감지하고 대응할 수 있도록 돕는 매커니즘.

원리: 
    이벤트 발생: 웹 애플리케이션에서 이벤트 발생
    웹훅 등록: 이벤트를 감지하고 대상 웹 애플리케이션(웹훅 리스너)의 엔드포인트 URL을 등록
    HTTP 요청 전송: 이벤트 발생 시, 웹 애플리케이션은 해당 이벤트와 관련된 정보를 포함해 등록된 웹훅 리스너로 HTTP POST 요청을 보낸다.
    이벤트 처리: 웹훅 리스너는 받은 HTTP 요청을 처리하고, 이벤트 작업을 수행한다. 이로 원격 애플리케이션에 실시간으로 이벤트의 발생을 알릴 수 있습니다.
예시:
    온라인 결제 시스템은 결제 완료 시 웹훅을 통해 주문 정보를 웹 애플리케이션으로 전달하여 주문 처리를 자동화할 수 있다.
    메시징 애플리케이션은 메시지 수신 시 웹훅을 사용하여 실시간으로 새 메시지를 다른 시스템으로 전달할 수 있다.

결론: 
    웹훅은 비동기적인 통신 방식으로 실시간 이벤트 전달을 가능하게 하며, 다양한 애플리케이션 간의 연동을 간편하게 구현할 수 있는 중요한 기술이다.


# Endpoint URL이란
특정 서비스나 에플리케이션 내에서 접근 가능한 기능이나 데이터의 위치를 나타내는 URL

예시:
    `http://localhost:8080/users`: 유저 정보를 조회하는 엔드포인트
    `http://localhost:8080/friends`: 친구 정보를 조회하는 엔드포인트
