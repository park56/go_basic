API 문서화
API작업을 항상 문서화 하고  최신으로 유지는 것은 팀원은 물론 다른팀과의 협업에도 매우 도움된다. 
이미 많은 자동 API문서 자동 빌드 애플리케이션/패키지 들이 많으므로 그것들을 사용하자.

REST 기반 API --
    Swagger -
        smartbear에서 디자인 되었다. 현제 거의 restful api의 문서화의 표준으로 간주되고 있다. Yaml 또는 json으로 작성된다.
        OpenAPI 는 restful api를 설명하기 위한 표준 형식이고 swagger는 이 형식을 구현하기 위한 도구와 프레임 워크 이다.
        https://swagger.io 참조
    
    API Blueprint -
        Apiary에서 디자인 되었다. MIT 라이선스. Markdown으로 작성된다.
    
    Raml -
        Rest API Modeling Language의 약자. HTTP의 구성 요소를 설명할 수 있다. Yaml형식으로 작성된다.
        https://raml.org 참조
